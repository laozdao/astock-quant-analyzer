# 架构设计文档

## 1. 概述

astock-quant-analyzer 是一个基于"双引擎四层"融合模型的 A股个股量化分析工具，旨在为投资者提供系统化、数据驱动的投资决策支持。

## 2. 系统架构

### 2.1 四层架构

```
┌──────────────────────────────────────────────┐
│            第四层：综合决策                     │
│   投资评级 · 买卖时机 · 仓位管理 · 催化因素      │
├──────────────────────────────────────────────┤
│            第三层：风险控制                     │
│   估值风险 · 流动性风险 · 下行风险 · 集中度风险   │
├──────────────────────────────────────────────┤
│  第二层：评分引擎                               │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐     │
│  │基本面八因子│ │ 量价六维度 │ │ 风险四维度 │     │
│  │  (60%)   │ │  (25%)   │ │  (15%)   │     │
│  └──────────┘ └──────────┘ └──────────┘     │
├──────────────────────────────────────────────┤
│  第一层：数据输入                               │
│  ┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐│
│  │基本面引擎│ │量价引擎 │ │板块引擎 │ │消息引擎 ││
│  └────────┘ └────────┘ └────────┘ └────────┘│
└──────────────────────────────────────────────┘
```

### 2.2 数据流

```
用户输入 → 数据采集 → 多维度评分 → 风险检测 → 综合决策 → 输出报告
```

## 3. 核心模块

### 3.1 数据输入层

负责从多个数据源获取股票相关信息：

- **基本面引擎**：财报数据、估值指标、行业数据
- **量价引擎**：行情数据、技术指标、资金流向
- **板块引擎**：行业板块、概念板块、板块表现
- **消息引擎**：公告、政策、新闻、研报

### 3.2 评分引擎层

#### 3.2.1 基本面八因子

| 因子 | 权重 | 说明 |
|------|------|------|
| F1 价值 | 15% | EP、PB、股息率 |
| F2 成长 | 15% | 营收增速、利润增速 |
| F3 质量 | 15% | ROE、毛利率、自由现金流 |
| F4 动量 | 10% | 价格动量、相对强度 |
| F5 情绪 | 10% | 舆情、分析师预期 |
| F6 资金流 | 10% | 主力资金、北向资金 |
| F7 ESG | 10% | 环境、社会、治理评分 |
| F8 另类数据 | 15% | 供应链、卫星数据等 |

#### 3.2.2 量价六维度

| 维度 | 权重 | 说明 |
|------|------|------|
| V1 趋势 | 25% | 均线排列、ADX |
| V2 量价配合 | 20% | 价量相关性 |
| V3 关键位置 | 20% | 支撑阻力、突破 |
| V4 主力行为 | 15% | 吸筹、洗盘、派发 |
| V5 波动率 | 10% | ATR、历史波动 |
| V6 多周期 | 10% | 日/周/月共振 |

#### 3.2.3 风险四维度

| 维度 | 权重 | 说明 |
|------|------|------|
| R1 估值风险 | 25% | PE/PB历史分位 |
| R2 流动性风险 | 25% | 成交额、换手率 |
| R3 下行风险 | 25% | 最大回撤、VaR |
| R4 集中度风险 | 25% | 行业集中度 |

### 3.3 风险控制层

- **信号冲突检测**：识别基本面与量价的背离
- **风险预警**：根据评分输出风险等级
- **仓位建议**：基于风险等级给出仓位上限

### 3.4 综合决策层

- **综合得分计算**：加权平均各维度得分
- **投资评级**：根据得分区间给出评级
- **经典框架适配**：匹配价值/成长/动量等策略
- **操作建议**：买入/卖出时机、止损止盈

## 4. 核心算法

### 4.1 综合得分计算

```python
综合得分 = 基本面得分 × 60% + 量价得分 × 25% + 风险得分 × 15%
```

### 4.2 动态权重调整

根据市场环境调整权重：

| 市场环境 | 基本面 | 量价 | 风险 |
|----------|--------|------|------|
| 牛市 | 50% | 35% | 15% |
| 震荡市 | 60% | 25% | 15% |
| 熊市 | 70% | 15% | 15% |
| 极端行情 | 40% | 10% | 50% |

### 4.3 信号冲突检测

```
IF 基本面 >= 7.0 AND 量价 <= 4.0:
    冲突 = "基本面强+量价弱：价值陷阱"
ELIF 基本面 <= 4.0 AND 量价 >= 7.0:
    冲突 = "基本面弱+量价强：垃圾股反弹"
ELIF 基本面 <= 4.0 AND 量价 <= 4.0:
    冲突 = "双弱：坚决回避"
ELIF 基本面 >= 7.0 AND 量价 >= 7.0:
    冲突 = "双强：最佳机会"
```

## 6. 性能考虑

- **数据缓存**：对不频繁变化的数据进行缓存
- **异步采集**：并行获取多个数据源
- **增量更新**：只更新变化的数据

## 7. 安全与合规

- **数据安全**：不存储用户敏感信息
- **免责声明**：明确标注不构成投资建议
- **风险提示**：充分揭示投资风险
